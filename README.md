PRACTICA GITHUB 2025/2026
Vamos a realizar en parejas un proyecto con una metodología SCRUM en el que la gestión de ramas va a ser la siguiente.
La rama main solo se utilizará para las entregas finales. La reléase o versionado del proyecto será de la rama main. Inicialmente se subirá de forma excepcional el esqueleto del proyecto a esa rama.
La nomenclatura para la creación de ramas es la siguiente:
SPRINTX/TIPOTAREA/001-DescripciónTarea
•	Evolutivo: feature
•	Corrección de errores: hotfix
Ejemplo de una rama:
Sprint1/feature/002NuevaPelicula
Temporalización de los SPRINTS.
